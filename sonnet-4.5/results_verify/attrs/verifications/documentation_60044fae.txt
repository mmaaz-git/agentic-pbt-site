Documentation Analysis
======================

I have reviewed both the inline documentation and the official attrs documentation.

1. Inline Documentation (attr/converters.py lines 125-151):
   The docstring explicitly states:
   - Values mapping to True: True, "true"/"t", "yes"/"y", "on", "1", 1
   - Values mapping to False: False, "false"/"f", "no"/"n", "off", "0", 0

   The documentation clearly specifies:
   - Integer 1 (not float 1.0) maps to True
   - Integer 0 (not float 0.0) maps to False
   - No mention of float values being accepted

2. Official Documentation (attrs.org):
   The official documentation at https://www.attrs.org/en/stable/api.html#converters confirms
   the same values as the inline documentation:
   - True values: True, "true"/"t", "yes"/"y", "on", "1", 1
   - False values: False, "false"/"f", "no"/"n", "off", "0", 0
   - No mention of floats 0.0 or 1.0

3. Documentation clarity:
   - The documentation is explicit and unambiguous
   - It lists specific values that are accepted
   - The numeric values are clearly shown as integers (0, 1) not floats
   - The string representations "0" and "1" are separate from the integer values
   - Any value not in these lists should raise ValueError

Conclusion: The documentation clearly states that only integer values 0 and 1 are accepted as
numeric inputs. Float values 0.0 and 1.0 are not documented as valid inputs and should raise
ValueError according to the documented contract.
