## Documentation Findings for dask.dataframe.io.orc.read_orc

### Function Signature and Parameters

The `read_orc` function is documented in the dask.dataframe.io.orc.core module with the following signature:

```python
def read_orc(
    path,
    engine="pyarrow",
    columns=None,
    index=None,
    split_stripes=1,
    aggregate_files=None,
    storage_options=None,
)
```

### Parameter Documentation

**columns**: None or list(str)
- Documentation states: "Columns to load. If None, loads all."
- This parameter controls which columns from the ORC file are loaded into the resulting DataFrame

**index**: str
- Documentation states: "Column name to set as index."
- This parameter specifies which column should be used as the DataFrame index

### Key Documentation Observations

1. **No explicit statement about columns/index relationship**: The documentation does NOT explicitly state whether the index column must be included in the columns parameter when both are specified.

2. **No examples showing combined usage**: The documentation provides only a basic example that doesn't demonstrate using both `columns` and `index` parameters together:
   ```python
   df = dd.read_orc('https://github.com/apache/orc/raw/master/examples/demo-11-zlib.orc')
   ```

3. **Reasonable user expectation**: Given the separate parameters for `columns` and `index`, a reasonable interpretation is that:
   - `columns` specifies which data columns to load
   - `index` specifies which column becomes the index
   - These are orthogonal concerns - the index column should be loaded automatically if needed

4. **Comparison with similar functions**: Other pandas/dask IO functions (like read_csv, read_parquet) typically handle index columns automatically even when not explicitly included in the columns parameter.

5. **Implicit behavior expectation**: The documentation's separation of these parameters implies they serve different purposes. If the index had to be included in columns, the documentation should explicitly state this requirement or provide a warning.

### Conclusion

The documentation does not specify the requirement that an index column must be included in the columns list. The absence of this specification, combined with the separate parameter design, suggests users should reasonably expect the function to handle loading the index column automatically when needed.
