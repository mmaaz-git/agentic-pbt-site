Documentation Analysis for numpy.random.Generator.dirichlet
============================================================

Official NumPy Documentation Findings:
--------------------------------------

From the NumPy documentation at https://numpy.org/doc/stable/reference/random/generated/numpy.random.Generator.dirichlet.html:

1. Parameter Requirements:
   - The alpha parameter is described as "Sequence of positive floats representing concentration parameters"
   - The documentation states that all values in alpha must be positive
   - It explicitly says the function "Raises ValueError if any alpha value is less than zero"

2. Mathematical Properties of Output:
   - The function generates samples from the Dirichlet distribution
   - Each sample produces a vector where all x_i > 0
   - The sum of all x_i must equal 1 (simplex constraint)
   - The output represents a probability distribution over a simplex

3. Computational Method:
   - The implementation generates samples by transforming standard gamma-distributed random variables

Mathematical Definition (Wikipedia):
------------------------------------

From the mathematical definition of the Dirichlet distribution:

1. Parameter Constraints:
   - ALL alpha values must be strictly positive: α_i > 0 for all i
   - The distribution is mathematically undefined for zero or negative alpha values
   - This is a fundamental mathematical requirement, not an implementation choice

2. Output Properties:
   - The output must lie on the (k-1)-simplex
   - This means: sum(x_i) = 1 and all x_i >= 0
   - This is a defining characteristic of the Dirichlet distribution

Key Documentation Observations:
-------------------------------

1. The NumPy documentation clearly states that alpha values must be positive
2. It says ValueError is raised for alpha < 0, but doesn't explicitly mention alpha = 0
3. However, mathematically, the Dirichlet distribution requires α_i > 0 (strictly positive)
4. The documentation describes the parameter as "positive floats" which should exclude zero

Comparison with Related Functions:
----------------------------------

The beta distribution (rng.beta), which is the bivariate case of Dirichlet:
- Correctly rejects zero parameters with ValueError: "a <= 0"
- This shows NumPy's intent to validate against non-positive parameters

Conclusion from Documentation:
-----------------------------

The documentation indicates that alpha parameters should be positive (excluding zero), though it could be more explicit about zero values. The mathematical definition is clear that zero is not allowed. The fact that the documentation says "positive floats" and that the related beta function rejects zero parameters suggests that dirichlet should also reject zero parameters.
