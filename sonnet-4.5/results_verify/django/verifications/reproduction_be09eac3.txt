REPRODUCTION ANALYSIS
=====================

I have successfully reproduced the bug reported in django.templatetags.i18n.GetLanguageInfoListNode.get_language_info.

Test Results:
-------------

1. Empty String Test:
   - Input: ''
   - Expected (per bug report): IndexError: string index out of range
   - Actual: IndexError: string index out of range ✓

2. Empty List Test:
   - Input: []
   - Expected (per bug report): IndexError
   - Actual: IndexError: list index out of range ✓

3. Empty Tuple Test:
   - Input: ()
   - Expected (per bug report): IndexError
   - Actual: IndexError: tuple index out of range ✓

4. Hypothesis Property-Based Test:
   - The test fails on empty string input as expected
   - The crash occurs at line 43 in i18n.py: `if len(language[0]) > 1:`

Root Cause:
-----------
The function attempts to access `language[0]` without first checking if the sequence/string is empty. When passed an empty string, list, or tuple, this causes an IndexError.

The proposed fix adds a check `if language and len(language[0]) > 1:` which would prevent the IndexError. With the fix:
- Empty string '' would result in: KeyError: 'Unknown language code .'
- Empty list [] would result in: KeyError: 'Unknown language code [].'
- Empty tuple () would result in: KeyError: 'Unknown language code ().'

These are more appropriate errors from the translation.get_language_info function that indicate the actual problem (invalid language code) rather than a low-level IndexError.
