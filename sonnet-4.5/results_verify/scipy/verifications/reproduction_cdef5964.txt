## Bug Reproduction Report

### Test Environment
- scipy version: 1.16.2
- Python: 3.13
- NumPy: installed

### Bug Report Verification

The bug report claims that `NumericAttribute._basic_stats` raises `ZeroDivisionError` when called with an array containing a single element.

### Test Results

1. **Hypothesis Test**: Successfully reproduced the bug
   - The hypothesis test with `@given(st.floats(allow_nan=False, allow_infinity=False))` immediately found the failing case
   - Any single float value triggers the ZeroDivisionError

2. **Direct Reproduction**: Confirmed the exact error
   - Code: `NumericAttribute('test')._basic_stats(np.array([5.0]))`
   - Result: `ZeroDivisionError: float division by zero`

3. **Root Cause Analysis**:
   - Line 227 in _arffread.py: `nbfac = data.size * 1. / (data.size - 1)`
   - When data.size = 1, this becomes: `1.0 / (1 - 1) = 1.0 / 0`
   - This causes the division by zero error

### Conclusion
The bug is **CONFIRMED** and reproducible. The error occurs exactly as described in the bug report. Any single-element numpy array passed to `_basic_stats` will cause a ZeroDivisionError.
