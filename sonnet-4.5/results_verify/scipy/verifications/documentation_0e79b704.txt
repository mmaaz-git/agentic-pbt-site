DOCUMENTATION ANALYSIS
======================

1. Exponential Model Documentation (_ExponentialModel class)
   - Defines model as: y = β₀ + e^(β₁x)
   - Provides example showing successful fitting
   - Example uses beta0=-10.0, beta1=0.5 and achieves perfect fit
   - DOES NOT specify accuracy expectations or guarantees
   - DOES NOT mention limitations of automatic parameter estimation

2. ODR Class Documentation
   - beta0 parameter is OPTIONAL
   - States: "Optional if model provides an 'estimate' function"
   - Exponential model DOES provide an estimate function (_exp_est)
   - Documentation implies the estimate function should work reasonably

3. Model Estimation Contract
   - The Model class expects estimate functions to provide reasonable initial values
   - Other models (linear, polynomial) have comments about returning "all ones"
   - The exponential model's _exp_est has comment "# Eh." suggesting known inadequacy

4. User Expectations vs Reality
   - Documentation example shows perfect fitting (SSR = 0.0)
   - This sets expectation that the model should fit exponential data well
   - No warning that automatic estimation may fail for certain parameter ranges
   - No documentation that users should provide initial estimates for reliability

5. Documentation Gaps
   - No mention that default estimation is unreliable
   - No guidance on when to provide initial parameters
   - No warning about potential convergence to poor local minima
   - Example happens to work, giving false confidence

Conclusion: The documentation sets an expectation of automatic fitting capability
through its example and the optional nature of beta0, but does not warn users
about the severe limitations of the default parameter estimation.
