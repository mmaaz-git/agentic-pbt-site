## Documentation Analysis for RangeIndex.linspace

### RangeIndex.linspace Documentation (xarray)

From the source code docstring (lines 227-277 of range_index.py):
- The method is documented to mimic numpy.linspace functionality
- Parameters documented:
  - `num`: "Number of values in the interval, i.e., dimension size (default: 50)"
  - `endpoint`: "If True (default), the stop value is included in the interval"
- The documentation does NOT specify any restrictions on `num` values
- The documentation does NOT state that `num` must be greater than 1
- The examples only show num=5, not edge cases like num=1

### numpy.linspace Documentation

From the official numpy documentation:
- The `num` parameter "Must be non-negative"
- No explicit restriction that num must be > 1
- Testing confirms numpy.linspace(0, 1, num=1, endpoint=True) returns [0.] without error
- Testing confirms numpy.linspace(0, 1, num=1, endpoint=False) returns [0.] without error

### Key Documentation Finding

The xarray RangeIndex.linspace claims to be "similar to numpy.linspace" (line 119-120 in the class docstring). Since numpy.linspace accepts num=1 without error, and the xarray documentation doesn't specify that num must be > 1, users would reasonably expect RangeIndex.linspace to also accept num=1.

The implementation detail that causes the crash (line 283) is an internal calculation that assumes num > 1 when endpoint=True, which is not documented as a requirement.
